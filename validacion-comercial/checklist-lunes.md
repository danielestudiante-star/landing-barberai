# Checklist lunes - Validacion comercial BarberAI

Usar esta lista para revisar BarberAI como si fueras un cliente antes de mostrarlo en una demo. Marcar cada punto despues de probar el deploy y anotar cualquier hallazgo en `bitacora-lunes.md`, `mejoras-antes-demo.md` o `backlog.md`.

## Revision general del sitio

- [ ] La pagina carga correctamente.
- [ ] El hero se ve profesional.
- [ ] El mensaje principal se entiende en menos de 10 segundos.
- [ ] Los botones principales funcionan.
- [ ] No hay imagenes rotas.
- [ ] No hay textos cortados.
- [ ] El flujo principal se puede mostrar en una demo.

## Revision del chatbot

- [ ] El chatbot abre correctamente.
- [ ] El chatbot permite escribir una pregunta.
- [ ] El chatbot responde preguntas sobre citas.
- [ ] El chatbot responde preguntas sobre horarios.
- [ ] El chatbot responde preguntas sobre servicios.
- [ ] El chatbot responde preguntas generales sobre la barberia.
- [ ] Las respuestas son claras para un cliente no tecnico.
- [ ] Si el chatbot falla, el error queda anotado en `backlog.md`.

## Revision de WhatsApp

- [ ] El boton de WhatsApp aparece visible.
- [ ] El boton de WhatsApp abre correctamente.
- [ ] El enlace abre el numero o chat esperado.
- [ ] El mensaje precargado, si existe, se entiende.
- [ ] El flujo de contacto por WhatsApp se puede explicar en una demo.

## Revision movil

- [ ] El sitio se ve bien en celular.
- [ ] El menu o navegacion funciona en celular.
- [ ] Los botones se pueden tocar sin dificultad.
- [ ] El chatbot se puede usar en celular.
- [ ] El boton de WhatsApp no tapa contenido importante.
- [ ] No hay desbordes horizontales.

## Revision de links

- [ ] Los links internos llevan a la seccion correcta.
- [ ] Los links externos abren correctamente.
- [ ] No hay links rotos.
- [ ] Los CTA principales llevan al siguiente paso esperado.

## Evidencia/capturas

- [ ] Captura del hero en desktop.
- [ ] Captura del sitio en celular.
- [ ] Captura del chatbot abierto.
- [ ] Captura de una respuesta util del chatbot.
- [ ] Captura del boton o flujo de WhatsApp.
- [ ] Captura de cualquier error encontrado.

## Resultado final

- [ ] El sitio esta listo para mostrarse en una demo.
- [ ] El sitio necesita ajustes menores antes de demo.
- [ ] El sitio necesita correcciones importantes antes de demo.
- [ ] Los pendientes quedaron documentados en `backlog.md`.
- [ ] La bitacora del dia quedo lista para enviar.
